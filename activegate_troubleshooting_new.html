üî¥ ActiveGate Not in UI (Not Connecting)

1
Verify ActiveGate Status
  • Confirm ActiveGate is not visible in Dynatrace deployment status screen
  • Check ActiveGate status in monitoring dashboards/alerts
  • Verify no recent maintenance windows or changes
2
Check ActiveGate Logs
Search Keywords: ERROR, Connect, Refused, TLS, Auth
Look For: "failed to connect", HTTP 401/403, SSL handshake errors, no heartbeat
  • Search Splunk/centralized logs for "failed to connect" messages
  • Look for HTTP 401/403 authentication failures in log aggregation
  • Check for SSL handshake errors or TLS issues
  • Verify heartbeat communication patterns
3
Resolution Actions
  • Document all findings from log analysis
  • Create escalation ticket with detailed symptom description
  • Include specific error messages found in logs
Success Indicator (post-escalation): Watch for "Connected to cluster" message in logs after infrastructure team applies fixes
Escalation Should Include: Connection failure log excerpts, deployment status snapshots, recent change details

‚ö†Ô∏è ActiveGate Connected, 0 Agents

1
Validate Status
  • Confirm ActiveGate appears in deployment status screen
  • Verify agent count shows as 0
  • Check ActiveGate connection status (should be green/connected)
2
Check Logs & Agent Communication
Search Keywords: listening, agent, connection, handshake
Look For: "Listening on 9999", inbound connection attempts (or lack thereof)
  • Search centralized logs for "Listening on 9999" message
  • Look for any inbound connection attempts from agents in logs
  • Check for handshake errors or certificate mismatches in aggregated logs
  • Review OneAgent logs from sample hosts for connection attempts
3
Resolution Actions
  • Verify OneAgent network zone configuration via Dynatrace UI
  • Check Dynatrace deployment status for agent assignments
  • Document log findings and create detailed escalation ticket
Success Indicators (post-escalation): "Listening on 9999" in logs, agent count > 0 in Dynatrace UI
Escalation Should Include: Log excerpts showing agent connection attempts, network zone settings, and any certificate errors

üìä Agent Count Inconsistent

1
Baseline Comparison
  • Determine expected agent count for this network zone
  • Compare current count with historical data
  • Check if any recent infrastructure changes occurred
2
Log Analysis
Analysis Scope for Production Support:
‚Ä¢ Dynatrace UI: Current vs historical agent counts
‚Ä¢ Monitoring Dashboards: Infrastructure change tracking
‚Ä¢ Log Aggregation: Pattern analysis across multiple ActiveGates
Search Keywords: disconnect, timeout, error, failover
Look For: Frequent connect/disconnect messages, error bursts, load warnings
  • Search centralized logs for frequent connect/disconnect patterns
  • Check for error bursts or timeout patterns in log aggregation
  • Look for load warnings across ActiveGate instances
  • Identify failover patterns between multiple ActiveGates
3
Resolution & Analysis
Note: Load balancing/failover between multiple ActiveGates is expected behavior
  • Document disconnect patterns and frequency from log analysis
  • Compare current behavior with historical baselines
  • Verify agent network zone assignments via Dynatrace UI
  • Create escalation ticket with detailed analysis and recommendations
Escalation Should Include: Agent count trends, disconnect frequency, resource usage patterns, suspected capacity issues
Success Indicator: Agent count remains stable with no recurring disconnect errors
